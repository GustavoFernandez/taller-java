Pruebas unitarias con

JUnit es un framework para realizar y automatizar pruebas de aplicaciones Java. Especificamente automatiza la ejecución de pruebas unitarias.

Pruebas Unitarias

Una prueba unitaria es un método que prueba una unidad de código. Al hablar de una unidad de código nos referimos a un requerimiento. Generalmente un requerimiento o funcionalidad se desarrolla en una porción de código delimitada por un método.

Las pruebas unitarias deben ser:

  • Independientes: No debe depender ni afectar a otras pruebas unitarias.
  • Automatizable: La prueba no debería requerir intervención manual.
  • Repetible y predecible: No debe incidir el orden y las veces que se repita la prueba, el resultado siempre debe ser el mismo.

Ventajas:

  • Fomentan el cambio: Facilitan la refactorización y modificación segura del código puesto que permiten hacer pruebas sobre los cambios y verificar que las modificaciones no han introducido errores.
  • Simplifican la integración: Permiten llegar a la fase de integración asegurando que las partes individuales funcionan correctamente. De esta manera se facilitan las pruebas de integración.
  • Menos errores y más fáciles de localizar: las pruebas unitarias reducen la cantidad de errores y el tiempo en localizarlos.

JUnit y Maven

Todo proyecto maven viene incorporado con el plugin surefire, este plugin tiene un goal llamado test que está enlazado a la fase test del ciclo de vida default. El plugin surefire puede trabajar por defecto con los frameworks de pruebas JUnit y TestNG.

El goal test ejecuta todas las pruebas unitarias en la fase de test del ciclo de vida default.

Agregando JUnit a nuestro proyecto


<dependency>
	<groupId>junit</groupId>
	<artifactId>junit</artifactId>
	<version>4.12</version>
	<scope>test</scope>
</dependency>

Anotaciones

@Test

Se usa para indicar que un método será ejecutado como una prueba de JUnit.

@BeforeClass y @AfterClass

Solo debe haber un método de cada una de estas anotaciones por clase, el método @BeforeClass es invocado una vez cuando inician las pruebas de la clase y el @AfterClass una sola vez cuando culminan todas las pruebas de la clase.

@Before y @After

Estos métodos se ejecutan antes (@Before) y después (@After) de cada método de prueba de una clase.

@Ignore

Los métodos marcados con esta anotación no serán ejecutados. Se suelen colocar para desactivar las pruebas que no pueden ser realizadas por algún motivo.

Aserciones

Una aserción es una afirmación de que un método debe ejecutarse y retornar resultados de la manera esperada.

assertEquals

Verifica que dos primitivos/objetos son iguales, en el caso de objetos utiliza internamente el método equals

assertTrue

Verifica que una condición sea verdadera

assertFalse

Verifica que una condición sea falsa

assertNotNull

Verifica que un objeto no sea null

assertNull

Verifica que un objeto sea null

assertSame

Verifica que dos referencias aputen hacía el mismo objeto. Internamente utiliza el operador de comparación '==' en los objetos pasados por parámetro.
